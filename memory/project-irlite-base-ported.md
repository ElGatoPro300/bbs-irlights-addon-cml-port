IRLite base ported into IRL-redactor — BBS-free (2026-06-16). Extends the standalone ImGui editor prototype (origin -> _archive/project-irl-redactor-imgui-standalone.md). The IRLite runtime lighting engine (the "база": collect lights -> bake shadows -> upload SSBO binding 7 -> inject Iris samplers) was ported from IRLite (C:\Users\Qualet\Documents\Project\Minecraft\BBS\bbs-irlights-addon, package qualet.irlite, a BBS addon) into IRL-redactor (...\BBS\irlights, package org.qualet.irlredactor, MC 1.20.4) and COMPLETELY decoupled from BBS. Source-of-truth for engine internals is still the rich IRLite memory (its addon-*, shader-*, patcher docs).

Status: gradlew build passes (jar + mixin refmap, ZERO BBS — BBS is not even a dependency); runClient boots to the title screen with no Mixin apply failed / mchorse / NoClassDefFoundError. Engine is correctly dormant at menu (Iris: "no valid shaderpack"). (Historical baseline note: the collect->bake->flush pipeline, the World mixins and the Iris-sampler mixins were not yet exercised at this initial milestone; later milestones exercise them in-world — see [[project-port-1211]].)

Package layout in IRL-redactor (src/client/java/org/qualet/irlredactor/):
- light/ — LightRegistry (the seam: registerPoint/registerSpot in, flush() out), LightBuffer (std430 SSBO binding 7), IrisShadersState (Iris-off gate), LightConfig (plain replacement for IRLite's BBS-backed IrliteConfig; static fields + same getters/defaults), and the NEW neutral source: PlacedLight (world-space POJO, stable long id), LightScene (static list the editor fills), LightDriver (replaces IRLite LightCollector — iterates LightScene -> registry each frame). (Note: LightRegistry/LightBuffer later migrated into irl-core — [[addon-light-buffer-ssbo]].)
- light/shadow/ — ShadowBaker, ShadowRenderer (BBS occluder/caster paths surgically removed), plus the PURE classes moved as-is: BlockShadowCollector/Cache/Entry, SpotlightDepthAtlas, PointShadowArray, IRLShadowQuality, ShadowBakeState. (Note: shadow orchestration later extracted to irl-core — [[plan-irl-core-shadow-extraction]].)
- mixin/client/ — GameRendererLightMixin (drives collect->bake->flush at renderWorld HEAD, LightDriver instead of LightCollector), WorldBlockChangeMixin, WorldBlockEntityTickersAccessor (the IRLite bbs. subpackage dropped — it targets vanilla World); mixin/client/iris/ ProgramSamplersBuilderMixin + SamplerBindingCubeArrayMixin. Config: src/client/resources/irl-redactor.client.mixins.json (registered in fabric.mod.json).

The seam (how light flows, BBS-free): editor -> LightScene.add(PlacedLight) -> GameRendererLightMixin each frame -> LightDriver.collect() -> LightRegistry.registerPoint/registerSpot (world coords, normalized +Z dir, cone cosines cos(toRadians(angleDeg*0.5)), stable id) -> ShadowBaker.bake() (writes shadow tiles back) -> LightRegistry.flush() (SSBO upload). Dormant when IrisShadersState.shadersDisabled().

What was DROPPED (this is the "BBS part", intentionally NOT ported): both BBS forms (PointLightForm/SpotlightForm), all form renderers, the BBS UI editor panels, BBS settings/L10n, all 8 BBS-targeting mixins + FilmsAccessor. In ShadowBaker/ShadowRenderer the BBS occluder/caster paths (model-block + film-replay collection; MorphRenderer/FormRenderer draw) were excised — so only vanilla entities + world blocks cast shadows now; the CASTER_MODEL_BLOCK/REPLAY constants are kept (harmless, still referenced by the cull math which never sees those types).

build.gradle deltas: added Modrinth + maven.gegy.dev repos; modImplementation "maven.modrinth:iris:1.7.2+1.20.4", modRuntimeOnly sodium mc1.20.4-0.5.8 + org.anarres:jcpp:1.4.14 + io.github.douira:glsl-transformer:2.0.0-pre13; added a processResources expand for ${version} in fabric.mod.json (pre-existing latent bug, fixed). No LWJGL pin (1.20.4). NO bbs dependency.

EDITOR (ImGui) — feature-complete (this file is the editor canon; origin = standalone ImGui prototype + wire-to-pipeline phases A–E, BOTH done, done-logs in _archive/project-irl-redactor-imgui-standalone.md + _archive/plan-wire-editor-to-pipeline.md). Editor opens with key L (detached-overlay path also works in Replay Mod — [[project-editor-vs-replay-screen-conflict]]). Pieces:

- Placement: X/Y/Z drag, "Переместить сюда" (player eye) / "Навести по взгляду" (look dir) + ImGuizmo 3D gizmo (translate for point, translate+rotate for spot), MC view/proj reconstructed per frame, NO new mixin. - Engine settings: engineGroup() <-> LightConfig statics (quality/cache/blocks/radius/guides) + auto-light controls ([[project-auto-block-lights]]) + cookie/gobo ([[project-spotlight-gobo-cookie-plan]]) + Патчер button (engine in irl-core, [[patcher]]).

How to actually SEE the light (test recipe): the engine only runs with Iris shaders ON, and a light is only VISIBLE if the active shaderpack contains the IRLite GLSL inject. Because LightBuffer (SSBO binding 7) + the Iris sampler mixins were ported byte-for-byte, a pack already patched by the original IRLite reads exactly what our engine writes -> it lights up. CONFLICT: do NOT run alongside irlite-0.0.1.jar (both write binding 7 / bind the same samplers). Clean test = Prism 1.20.4 + Iris + Sodium + a patched pack + irl-redactor-1.0.0.jar, WITHOUT the irlite jar; in-world press J. Jar at build/libs/.
