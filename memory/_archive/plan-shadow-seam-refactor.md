# Вынос теней за шов ShadowCasterSource

Это продолжение [[project-irl-sync-strategy]]: Ф0+Ф2 (общий `irl-core` = патчер + ядро света) уже сделаны и закоммичены. Здесь — следующий большой шаг: **тени**. Утверждено пользователем 2026-06-17 после grounded-анализа кода (2 агента читали `ShadowBaker`+`ShadowRenderer` обоих модов).

## ПРОСТОЕ ОБЪЯСНЕНИЕ ДЛЯ ПОЛЬЗОВАТЕЛЯ (использовать дословно при отчётах)

> Тени отличались по двум причинам: (1) аддон умеет рисовать тень по геометрии **BBS-формы актёра** (кастомный морф — как в BBS-превью), а редактор рисует то, что есть в ванильном мире — **настоящие ванильные модели мобов/блоков** (НЕ коробку!). Важно: на 1.20 редактор даёт правильный силуэт; «коробка» от энтити — это регрессия ПОРТА 1.21.11 (там render-API переписан на raw-GL и энтити временно упрощён до bounding-box), а не поведение редактора как такового. (2) редактор просто свежее по оптимизациям.
> Идея: **«как нарисовать тень от каста» — это единственная BBS-завязка**, выносим её за интерфейс (`ShadowCasterSource`) — как мы уже сделали с патчером (`PatcherHost`). Аддон подставляет «рисуй форму», редактор — «рисуй бокс». **Вся остальная логика теней (раскладка карт глубины, планировщик, оптимизации) — общая.** Бонус: при общей оркестрации IRLite **получает все 6 оптимизаций даром** — откатывать/бэкпортить вручную не нужно.
> Схема на одной версии: одно ядро ↔ аддон (BBS-каст) + редактор (бокс-каст). Ядро BBS-free, поэтому совместимо с обоими.

## АРХИТЕКТУРА (утверждена)

```
shared (оркестрация теней, MC-типизирована)
  ├─ раскладка spot-атласа / point-cube-array  (классы SpotlightDepthAtlas, PointShadowArray — УЖЕ отдельные)
  ├─ цикл по источникам, отсечение конус/грань, назначение тайлов (sticky)
  ├─ dirty-кэши + 2-слойный бейк (static base → GPU-копия → overlay)
  └─ ВСЕ 6 перф-опт (T1.1/T1.2/T1.3/T2.2/T2.4/T2.5)
        ▲ вызывает только интерфейс ▼
   ShadowCasterSource (шов)
        ├─ аддон IRLite  → рисует BBS-форму актёра (FormRenderer/MorphRenderer)
        └─ редактор      → 1.20: НАСТОЯЩАЯ vanilla-модель энтити (EntityRenderDispatcher) + блоки (BakedModel)
                            1.21.11(порт): бокс/блоб (flushCasterBoxes) — РЕГРЕССИЯ, кандидат на починку через шов
```
Каст-блоков (cutout/AABB) BBS-free → может быть ОБЩИМ дефолтом, не частью BBS-шва.
ВАЖНО (не перепутать): на 1.20 редактор НЕ рисует коробку от энтити — рисует реальную ванильную модель (ShadowRenderer.drawEntity → EntityRenderDispatcher.render). Коробка от энтити есть ТОЛЬКО на порте 1.21.11 (ShadowRenderer.flushCasterBoxes, «interpolated bounding box / blob occluder», version-robust заглушка после raw-GL переписки). BBS-шов = не «модель vs коробка», а «умеет рисовать BBS-морф актёра vs рисует ванильную модель мира». Шов ПОЗВОЛЯЕТ позже вернуть реальные модели энтити и на 1.21.11 (переписать caster-impl редактора под новый render-API) — локально, не трогая оркестрацию.

## КОНТРАКТ ШВА (черновик — заморозить в Фазе 1)

```java
interface ShadowCasterSource {
    // Заменяет collect(): заполняет SoA окклюдеров через sink (без боксинга — SoA фикс. 32).
    // Должен выставить: центр+радиус ограничивающей сферы (для cull), dynamic-флаг, staticHash.
    void collect(ClientWorld world, Vec3d camPos, float tickDelta, OccluderSink sink);
    // Заменяет лист renderCaster/bufferCaster. Кадр УЖЕ выставлен (FBO/viewport/scissor/матрицы
    // light view+proj на RenderSystem, depth-state, открыт batch). Имплементация только эмитит геометрию.
    void drawOccluder(Object caster, int type, float tickDelta, Immediate batch, Matrix4f view, Matrix4f proj);
}
```
Интерфейсу НЕ нужны: FBO/тайл/индекс источника/cutout-флаг — это владеет shared `beginSpot/beginPointFace`. НУЖНО: tickDelta, общий batch Immediate, и (несущее) view+proj (их явно передают в VertexBuffer.draw, чтобы обойти порчу modelview). Блок-каст (cutout/AABB) — отдельный общий хелпер `drawBlockOccluders(id, blocks)` или внутри shared, НЕ часть BBS-шва.

### 5 ИНВАРИАНТОВ (несущие; провал НЕ даёт ошибку компиляции — только глазами в мире)
1. **Порча матриц.** Каст через ванильный EntityRenderer пачкает RenderSystem modelview. Shared-слой ОБЯЗАН переустановить view/proj перед блок-пассом (как сейчас `flushCasterBatch`), иначе **тени блоков молча исчезают**. Контракт: «drawOccluder может испортить живые матрицы; shared переустанавливает их перед блоками; VBO-пути рисуют через переданные view/proj».
2. **occType static/dynamic — несущий.** `CASTER_MODEL_BLOCK == static` (вшивается в never-rebaked базовый слой + сигнатуру); всё остальное == dynamic (перерисовка каждый кадр). Мис-тег движущейся формы как static → её тень **замёрзнет** в static-тайле.
3. **staticHash-контракт.** Неверный хэш → устаревшая тень держится через пропущенные кадры (sticky-тайлы усиливают). BBS-impl: form-identity + Transform scale/rotate (бывший modelBlockHash). Editor: box/blockstate-хэш. Для dynamic = 0.
4. **Изоляция исключений.** Per-caster try/catch ДОЛЖЕН остаться в shared-обёртке (сейчас renderCaster/bufferCaster ловит throw и флашит batch), иначе одна битая форма сливает мусор-квады в следующий каст / рушит бейк.
5. **Конвенция ограничивающей сферы.** center (поднят к середине высоты) + radius (полу-макс-ребро + OVERLAP_MARGIN) должны совпадать с тем, что ждёт cull (`insideCone`/`sphereTouchesFace` используют orad и orad*SQRT2). Чужая сфера → cull тихо роняет валидные касты.

## РОЛИ СУБ-АГЕНТОВ (роестр; переиспользуются между фазами)

- **R1 Seam-cartographer** — читает 2 shadow-файла ОДНОГО мода; выдаёт разбивку на 3 слоя (каст/оркестрация/GL) + якоря методов-швов. (Ф1; карты этой сессии можно подать как кэш.)
- **R2 Contract-author** — читает обе карты R1; черновик `ShadowCasterSource`+`OccluderSink`+список инвариантов. (Ф1)
- **R3 Invariant-auditor (адверсариальный)** — читает контракт + 5 рисков; пытается СЛОМАТЬ каждый инвариант сценарием каста; выдаёт дыры. (Ф1, Ф3)
- **R4 Refactorer** — читает shadow одного мода + spec; делает врезку шва, ПОВЕДЕНИЕ-сохраняющую. (Ф2, Ф3)
- **R5 Behavior-diff-reviewer** — читает до/после; подтверждает, что логика оркестрации не изменилась (двигали структуру, не поведение). (Ф2, Ф3, Ф4)
- **R6 In-world-verifier** — гоняет runClient; ставит сцену (лампа + движущаяся энтити + блоки + шейдерпак); проверяет: тени блоков есть, тень движущейся формы СЛЕДУЕТ (не замёрзла), нет мерцания, совпадение с BBS-превью (IRLite); репортит + помечает, что юзер обязан глянуть сам. (Ф0, Ф2, Ф3, Ф4)
- **R7 BBS-caster-impl-author** — читает BBS draw-пути IRLite (drawEntity/drawModelBlock/drawReplay) + spec; реализует `ShadowCasterSource` для BBS, удовлетворяя 5 инвариантам. (Ф3)
- **R8 Sharing-extractor/Build-wirer** — экстрагирует унифиц. оркестрацию; врезает composite/JiJ или Stonecutter; добивается зелёного билда. (Ф4, Ф5)
