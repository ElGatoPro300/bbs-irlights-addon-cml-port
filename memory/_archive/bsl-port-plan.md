Roadmap for porting the IRLite inject to BSL Shaders v10 (by Capt Tatsu), fourth target after IterationRP, Photon, ComplementaryReimagined. Parent: [[MEMORY]]. WHERE/HOW to hook = [[bsl-pipeline]] (read it first every phase). GROUND RULES (same as Photon/Complementary ports, user-set): work entirely in Shadres/Modification/BSL (clean copy of Shadres/Original/BSL made 2026-06-12, 285 files count-verified); ALL phases hand-edit Modification; diff + build the .irlpatch ONLY at the very end (Phase 5). Each phase = its own session. The Java addon and the patcher applier are pack-agnostic and DO NOT change. BSL+Complementary structural similarity is the porting lever — most lib halves move VERBATIM; the deltas are catalogued in [[bsl-pipeline]] "KEY DIFFERENCES".

INJECT FILE: shaders/lib/irlite/irlite_lights.glsl. Self-contained IRLITE_* defines at top; in-file `#extension GL_ARB_shader_storage_buffer_object : enable` (+ texture_cube_map_array + texture_gather in Phase 2). Halves: surface `#ifdef IRLITE_SURFACE_PASS` (defined by the forwardLighting hook; + !VOXY_PATCH !IRLITE_SKIP), VL `#ifdef IRLITE_COMPOSITE_PASS && IRLITE_VOLUMETRIC`, outline `#ifdef IRLITE_COMPOSITE_PASS && IRLITE_OUTLINE` (bare-#ifdef pairs for Iris option registration). ONE colour convention everywhere: RAW LINEAR (no pow(1/2.2) anywhere — BSL lights in linear space).

ESTIMATED FINAL OP INVENTORY (~26): +file ×8 (lib + program/deferred2.glsl + 6 deferred2 wrappers); forwardLighting ×3 (include-before-signature, compute-before-albedo-multiply, add-after-vanillaDiffuse-line — final body has bare #ifdef DIFFUSE/SPECULAR nesting); composite ×2 (include after the waterFog include, outline+VL-upsample block after the reflectionColor line); IRLITE_NONTERRAIN ×4 (entities/entities_glowing/hand/block, after #ifdef FSH); IRLITE_SKIP ×2 (dh_terrain/dh_water); shaders.properties ×6 (iris.features replace + #ifndef-MCBL insert, deferred2-toggles+size.buffer block before the MCBL comment, screen.LIGHTING replace, screens block after screen.AO, sliders tail replace); lang ×1. POST-PORT PERF — VL FULL-RES COST.  STATUS: RESOLVED — reduced-res rework VERIFIED in-game 2026-06-12 (user: "то что нужно"), COMMITTED 7ffd903. Complementary backport DONE 2026-06-12 (see [[complementary-port-plan]] FOLLOW-UP; committed c7e8e23 + patch regen 8f8805e). (Original report: FPS drops inside a light's radius with VL on, and even just looking at a beam from outside; Photon fine, Complementary same problem.)
- ROOT CAUSE (architectural, explains the Photon/others split): in Photon our VL rides the pack's own c0_vl pass which renders QUARTER-RES (1/16 of the pixels + bilateral upsample) — inherited for free. Complementary (composite1) and BSL (composite) have NO reduced-res stage — our march is FULL-RES: every pixel whose ray intersects the light volume runs IRLITE_VL_STEPS steps, each with HG phase + exp + (with IRLITE_VL_SHADOWS) a baked-map tap — and the POINT path taps a samplerCubeArray, the most expensive fetch in the inject. Inside the radius every screen pixel's ray is in the volume -> the whole screen marches; looking at a beam -> all covered pixels march (the rest early-out on the cone/sphere test, cheap). Cost ∝ covered pixels × steps.
- MITIGATIONS LANDED (lib + properties + lang; ~2.8x cheaper worst case): (1) zero-beam early skip — `if (dot(lightCol, lightCol) < 1e-8) continue;` BEFORE the intersection math (beamStrength-0 lights used to pay the FULL march for zero contribution — real waste bug, backport to Photon+Complementary too); (2) IRLITE_VL_STEPS default 14 -> 10, list extended down `[6 8 10 12 14 16 24 32]`; (3) NEW option IRLITE_VL_SHADOW_STRIDE 2 `[1 2 3 4]` — tap the shadow map every Nth step, cached shadowVis reused in between (dithered ray start hides the quantization; 1 = old behaviour); added to the VL screen + sliders + lang. Preprocessor balance 28/28.
- A/B TO CONFIRM DIAGNOSIS (user): toggle IRLITE_VL_SHADOWS off — if FPS mostly recovers, the cube taps dominate (stride/steps scale that part); remaining cost = the march itself (steps slider).
- REAL FIX LANDED 2026-06-12 (user verdict on mitigations: "это беда" — went straight to the architecture): REDUCED-RES VL PASS, Photon's c0_vl mechanism rebuilt for BSL. NO corner-trick needed — Iris `size.buffer` does it: a pass whose ONLY render target is a fraction-sized buffer gets that viewport (syntax verified against Photon's properties: `size.buffer.colortex10 = 0.5 0.5`).
  - NEW PASS: program/deferred2.glsl + 6 wrappers (world0/world1/world-1 × .fsh/.vsh — BSL has deferred/deferred1 in all 3 dims, deferred2 fits the pattern; wrappers carry OVERWORLD/END/NETHER + FSH/VSH). Runs at the deferred stage (after solid gbuffers — depthtex0 IS the opaque depth there, exactly the march end we always used; translucent-blindness irrelevant for VL by design). FSH: reconstructs playerPos from depthtex0 + gbufferProjectionInverse/ModelViewInverse (own uniform decls — our file), dither = pack's /lib/util/dither.glsl Bayer8 + frameCounter%8 temporal under TAA, `#define IRLITE_VL_PASS` + lib include, calls irlite_volumetric, writes `/* RENDERTARGETS: 10 */`. GOTCHA in its header comment: `world*\/deferred2` is escaped — a literal `*/` inside the block comment would terminate it. BUFFER-FORMAT GOTCHA (first compile, fixed): `const int colortex10Format = RGB16F;` as BARE CODE in deferred2 reached the GLSL compiler -> `C1503 undefined variable "RGB16F"` — BSL declares ALL buffer-format consts INSIDE a `/* ... */` block comment in final.glsl (the loader parses comments; the compiler never sees them). Our directive moved INTO that block, after the colortex9Format line (final.glsl gains one patch op; colortex10 was FREE — BSL uses 0-9 + a stray colortex16 entry).
  - LIB regated: VL half `#ifdef IRLITE_COMPOSITE_PASS` -> `#ifdef IRLITE_VL_PASS`; COMPILE_SHADOWS third term -> `(IRLITE_VL_PASS && VOLUMETRIC && VL_SHADOWS)` (composite no longer compiles the march or pulls cube samplers unless OUTLINE needs them); NEW upsample-side block `#ifdef IRLITE_COMPOSITE_PASS + #ifdef IRLITE_VOLUMETRIC -> uniform sampler2D colortex10;` (deferred2 must NOT see this decl — it renders to that buffer). NEW option `IRLITE_VL_RESOLUTION 50 // [100 50 25]`. Balance 30/30.
  - composite.glsl VL block: march REPLACED by `vec3 irliteVL = texture2D(colortex10, texCoord).rgb;` (bilinear upsample — colortex default LINEAR filter, same as Photon relies on) + the underwater/lava parity stays at the add site.
