Roadmap for porting the IRLite inject from IterationRP to the Photon shaderpack. Parent: [[MEMORY]]. WHERE/HOW to hook = [[photon-pipeline]]. The inject internals being ported = [[shader-irlite-glsl]] / [[shader-shadow-sampling]] / [[shader-volumetric]] / [[shader-settings]]. Folder/test loop = [[sync-workflow]]. Final output = patches/photon.irlpatch ([[patcher]]).

GROUND RULES (user-set, 2026-06-06): work entirely in Shadres/Modification (a clean copy of Shadres/Photon); do ALL phases by hand-editing Modification; take the diff and build the .irlpatch ONLY at the very end (Phase 5). Each phase = its own session to keep context small. The Java addon (Phase-1 pillar) and the patcher applier (Phase-2 pillar) are pack-AGNOSTIC and DO NOT change — only the GLSL inject + settings + the new patch file change.

**Why:** Photon is far larger/more complex than IterationRP; one session can't hold the whole port without context overflow.
**How to apply:** start each phase in a fresh session; read [[photon-pipeline]] + the relevant [[shader-*]] doc; edit only Modification; test via the sync loop; update this file's STATUS line and the shader docs when a phase lands.

DIRECTORY STATE (paths refactored 2026-06-11, see [[sync-workflow]]): Shadres/Original/Photon = pristine base (diff baseline, never edit; was Shadres/Photon). Shadres/Modification/Photon = working copy (full pack copy; was the flat Shadres/Modification). Test target = run/shaderpacks (copy Modification/Photon's changed files there, then runClient / hot-reload).

PROPOSED INJECT FILE: shaders/include/irlite/irlite_lights.glsl (new +file; Photon groups includes in subfolders). Keep all IRLITE_* #define options self-contained at its top (Iris scans GLSL for option #defines) so Phase 4 settings only touches shaders.properties + lang, not settings.glsl — matches the IterationRP inject and minimizes patch anchors.
