ShadowBaker (driver, qualet.irlite.client.light.shadow.ShadowBaker)
- bake(world, cameraPos, cameraForward, tickDelta): no-op if no lights. Applies quality preset (IRLShadowQuality.applyFromSetting(config.shadowQuality())). cameraForward (Vec3d.fromPolar(pitch,yaw) from GameRendererLightMixin) drives the BEHIND-CAMERA cull (audit #2, 2026-06-09 commit 181cf88): a light with dot(pos-cam,forward) < -range (whole influence sphere behind the camera plane) is skipped BEFORE scanInRange/collectBlocks — no bake, no CPU collect, no tile consumed, shadowTile stays -1 (never sampled off-screen), first-bake on return. Safe (only the behind half-space; never a side-of-frustum light). Calls ShadowRenderer.beginBake() once before the loops (audit #3 glGet snapshot, see ShadowRenderer below).
- collect() occluders into parallel arrays (max MAX_OCCLUDERS = 32, within COLLECT_DIST = 72 blocks):
    - world entities: LivingEntity + ItemEntity, via world.getEntities(); center = lerped render pos + half height; radius from bounding box + OVERLAP_MARGIN 0.5. casterType CASTER_ENTITY.
    - BBS model blocks: world.blockEntityTickers -> ModelBlockEntity (enabled), pos+transform; radius from form hitbox*scale. casterType CASTER_MODEL_BLOCK.
    - BBS film replays: BBSModClient.getfilms().controllers (FilmsAccessor) + the active editor controller (only when dashboard screen actually open, getActiveEditorController guards against the cached-but-closed dashboard). Skips replays where replay.actor (actors are real entities already). casterType CASTER_REPLAY.
- Cache (PER-LIGHT since 2026-06-09, commit ec120e5; replaced the old global additive sceneHash — see [[project-shadow-bake-perf-audit]] #1 for the full design + why age can't be an anim token). Keyed by getId(i): a light is dirty when first-baked, or an entity/replay is in range (dynamic subject -> re-bake every frame), or its own geometry changed (pos/dir/range/cosOuter/shadows), or an in-range model block changed form/moved/rotated/scaled (order-independent sum of model-block hashes; form identityHashCode folded in since sticky tiles), or its world-block list instance changed (terrain edit), or its assigned tile changed (insurance; near-impossible under sticky tiles), or wasDynamic (one trailing rebake after a subject leaves range). !config.shadowCache() forces always-dirty. If not dirty, skip the GL depth render and reuse last frame's depth maps (tiles/slots still assigned).
- STICKY TILES (perf-audit R2, 2026-06-09, after a29794b): tiles/slots are no longer sequential-per-frame. acquireTile(owner[],active[],id): a light OWNS its tile/slot across frames (kept while culled / toggled off / nothing in range); tileless light takes a free tile, else steals the most-stale owner inactive >= STALE_FRAMES=2 frames (victim's dirty state purged -> clean first-bake on return; the 2-frame floor stops same-frame robbery of a later-iterated owner, else two lights could ping-pong a tile re-baking both every frame), else -1 (unshadowed this frame). Consequences: a light dropping out (behind-camera cull, skip, removal) no longer shifts later lights' tiles (was: mass rebake on every camera pan); a returning owner with unchanged sig RE-USES its old map with ZERO rebake (safe: tile content can only be its owner's). Dirty state is retained while a light owns a tile (retainDirtyState now keeps tile OWNERS, not this-frame bakers; bakedIds removed); purge paths = steal, resetTileState (count==0 drain, shadow-quality preset change — the preset switch frees the textures, so trusting old sigs would sample blank maps; this also fixed a latent pre-existing bug). Spot/point loops now iterate all lights (no tile<MAX loop bound); per-loop budget enforced by acquireTile returning -1.
- TWO-LAYER BAKE (perf-audit R1, 2026-06-09): three per-light branches in bake(). (1) !cache -> legacy full bake into live every frame (CASTERS_ALL). (2) pure static (!dyn && !wasDynamic) -> bake CASTERS_STATIC + blocks into LIVE tile only when dirty (sig/blocks/tile/first-bake) — today's cache-hit behavior. (3) OVERLAY (dyn || wasDynamic): if hasStatic (static occluders or blocks): bake static content into the STATIC tile only when staticStale (own lastStaticSig/lastStaticTile/lastStaticBlocks maps, same key+eviction as live maps), then EVERY frame GPU-copy static->live (SpotlightDepthAtlas.copyStaticToLive(tile) = glCopyImageSubData rect; PointShadowArray.copyStaticToLive(slot) = ONE call copying all 6 layer-faces) and draw ONLY dynamic casters on top (begin* with clear=false; point: faces without a dynamic sphere touch are skipped — faceHasDynamic — the copy already refreshed them). If !hasStatic: clear + dynamics only (no static texture touched). Leave-frame (!dyn && wasDynamic): same flow draws no dynamics -> live = clean static; rememberLive() then marks live as pure-static and clears wasDynamic (the old "one trailing rebake" is now a copy). CONSEQUENCE: per-frame cost of a film scene = moving subjects only; cutout re-tessellation + model-block form-tree re-render + block VBO redraw happen ONLY on static change (was: every frame x6 faces with an actor in range). Caster filters CASTERS_ALL/STATIC/DYNAMIC (static = CASTER_MODEL_BLOCK; dynamic = entity+replay) on renderInRangeCone/renderInRangeFace; scanInRange also outputs staticInRangeScratch (count). Moving light (sig changes every frame) in overlay = static re-bake every frame + copy — slight overhead vs old combined bake, accepted (audit notes R3 quantized block cache as the moving-light fix).
- PROFILER: -Dirlite.profileShadows=true -> bake() wraps bakeInner with nanoTime; logs ~1/s: avg/max bake ms, static-bake counts (spot/point), overlay-frame counts (spot/point), occluder/light counts. Validation tool for the perf work ([[project-shadow-bake-perf-audit]]).
- SPOT pass: for each spot: skip if range tiny or (no occluders && no blocks); tile = acquireTile (sticky, -1 = skip); outer angle from acos(cosOuter)*2; if dirty: ShadowRenderer.beginSpot(tile,...) -> renderInRangeCone() -> renderBlocksDepth() -> endPass(); setShadowTile(i, tile). CONE CULL (audit #4 B11, commit 1378305): scanInRange + renderInRangeCone skip occluders fully outside the spot's lit cone (insideCone: phi-alpha>acos(cosOuter); provably safe — they can only shadow unlit fragments). The cull also drops out-of-cone occluders from the dirty signature/dynamic flag, so a subject outside the cone no longer re-bakes the spot each frame.
- POINT pass: for each point: same range/occluder check; slot = acquireTile (sticky); if dirty: for 6 faces beginPointFace(layer,face,...) -> renderInRangeFace(face) -> renderBlocksDepth() -> endPass(); setShadowTile(i, layer). PER-FACE CULL (audit #4 B10, commit 1378305): renderInRangeFace draws only occluders whose bounding sphere intersects that face's 90° frustum (sphereTouchesFace, conservative w/ r*sqrt2 margin) — ~5/6 fewer caster draws. All 6 faces are still begun/cleared/ended (a vacated face must clear or it keeps a phantom shadow); only the caster draw is per-face culled. The block VBO is still drawn into all 6 faces (B7 deferred to #6).
- scanInRange (count + sets dynamicInRangeScratch/staticOccSigScratch) and the renderInRange* helpers take ALL occluder types (the old per-type skip()/skipEntities plumbing was removed 2026-06-09). reach = lightRange + occluderRadius (sphere). Spots additionally pass dir+coneTheta+cone=true for the cone cull; points pass cone=false and cull per-face at render time. The per-light "shadows" master toggle (LightRegistry.getShadows, default true) is applied in bake() instead: when OFF, entInRange is forced to 0 AND blocks emptied, so the light drops into the "nothing in range" continue, its tile stays unassigned (-1 = none -> SSBO vlParams.w=-1 -> shader renders it unshadowed).
- BLOCK shadows (added 2026-06-06; full impl notes in archived shadow-entity-parity-findings #3): per in-range light, collectBlocks(id,...) returns world blocks via the identity-keyed BlockShadowCache.getOrCompute (BlockShadowCollector, radius clamped to IrliteConfig.shadowBlockRadius(), a BBS setting, default 24, range 4..96 — commit a29794b; was a hardcoded MAX_BLOCK_COLLECT_RADIUS=24); called every frame for tile-stability but O(1) on a hit. QUANTIZED (perf-audit R3, 2026-06-09): getOrCompute snaps the collection sphere (center Math.round per axis, radius ceil + 0.87 pad > cell half-diagonal) before hashing/collecting/section-indexing, so a continuously-moving lamp (entity-mounted / transform-animated) re-collects only on a 1-block cell crossing instead of every frame, and the stable list instance keeps the block VBO from rebuilding; padded blocks past range are clipped by the bake far plane. Skip is now (entInRange==0 && blocks.isEmpty()); point collects once before 6 faces; blocks AND entities are both gated by the per-light "shadows" master toggle (OFF = neither is baked; was: blocks ignored the old noEntityShadows flag). After the loops, retainOnly/retainBlockVbos(liveIds) evict caches for gone lights (and drain when the feature is off). WorldBlockChangeMixin: invalidateAt(pos) only (precise CPU re-collect) — the old markBlocksDirty() global force was REMOVED with the per-light cache (2026-06-09): invalidateAt makes the next getOrCompute return a NEW list instance, which ShadowBaker detects by reference and re-bakes just the affected lamp. Gated by setting shadow_blocks (default ON).

ShadowRenderer (depth passes, ShadowRenderer)
- beginSpot(tile,...): bind SpotlightDepthAtlas FBO, viewport+scissor to the tile, clear depth; proj = perspective(fov=outerDeg, aspect 1, near 0.05, far=range); view = lookAt(lightPos -> lightPos+dir, stable up). applyMatrices.
- beginPointFace(slot,face,...): PointShadowArray.bindFaceForRender(slot,face); viewport FACE_SIZE; proj = perspective(90deg, near 0.05, far=radius); per-face dir/up table (+X,-X,+Y,-Y,+Z,-Z). applyMatrices.
- renderCaster(caster,type,tickDelta): pins depthMask/depthTest/no-blend; ShadowBakeState.setBaking(true) around the draw; switch on type:
    - CASTER_ENTITY drawEntity: tries BBS MorphRenderer.renderPlayer / renderLivingEntity (so morphed actors bake their visible silhouette), falls back to vanilla EntityRenderDispatcher.render. FULL_LIGHT lightmap.
    - CASTER_MODEL_BLOCK drawModelBlock: FormUtilsClient.getRenderer(form).render with FormRenderingContext(MODEL_BLOCK,...).
    - CASTER_REPLAY drawReplay: same via FormRenderingContext(ENTITY,...), rotated by -bodyYaw.
  immediate.draw() after; a bad caster is swallowed (try/catch) so one failure doesn't abort the bake.
- renderBlocksDepth(long id, List): called inside a begin*/endPass bracket AFTER the entity casters. Cutout entries first (renderBlocksDepthCutout: BlockRenderManager.renderBlock per block -> immediate.draw, alpha-test shader drops transparent texels, NOT VBO-cached), then shaped entries via a per-id VBO cache (buildBlockVbo: Tessellator QUADS POSITION, QuadBoxConsumer over VoxelShape.forEachBox; rebuilt only when the list instance changes, i.e. on a cache miss; drawn via getPositionProgram). Self-balances cull. anyShape guard skips the AABB path when every entry is cutout (empty buffer would throw). releaseBlockVbo(id)/retainBlockVbos(liveIds) free GPU buffers.
- beginBake()/savePassState/endPass: save+restore FBO, viewport, scissor, projection matrix, vertex sorter, color write mask, modelview stack. Critical for not corrupting MC/Iris GL state. The save is the original MC/Iris state, invariant across all passes of one bake (each endPass restores it), so the ~5 glGet* (FBO/viewport/scissor/colormask = CPU<->GPU sync) are taken ONCE per bake, not per pass (audit #3, commit 181cf88): savePassState snapshots only when !passStateSaved; ShadowBaker.bake calls beginBake() (resets passStateSaved) once before the loops to re-arm. endPass still restores per pass from the single snapshot. WARNING: if beginBake() is ever not called, passStateSaved sticks true and the snapshot freezes (stale-state restore).
- applyMatrices: colorMask all true; set projection (VertexSorter.BY_DISTANCE); push modelview, load identity, multiply by view, applyModelViewMatrix.

ShadowBakeState — boolean flag, true during caster rendering. Light form renderers (AbstractLightFormRenderer.render3D) check it and skip light registration while baking (else a light form inside a caster's tree re-registers every face/tile pass).

TEXTURES (DUAL-LAYER since R1: each class owns a LIVE texture+FBO — what the shader samples, getGlTextureId() — and a STATIC texture+FBO for the overlay base; getFboId(boolean staticLayer) / bindFaceForRender(slot,face,staticLayer); copyStaticToLive via GL43.glCopyImageSubData; both freed by delete()/preset switch; the static layer allocates lazily on the FIRST overlay bake, so scenes with no dynamic casters near lamps pay no extra VRAM, otherwise VRAM doubles)
- SpotlightDepthAtlas: 2D depth atlas, GRID_X x GRID_Y = 4x4 = 16 tiles, TILE_SIZE (preset). Format DEPTH_COMPONENT32F, NEAREST, COMPARE_MODE NONE (manual compare in shader). tile i at pixel ((i%4)*TILE, (i/4)*TILE). Lazy alloc (getGlTextureId() returns 0 until first bake -> no VRAM when no spot).
- PointShadowArray: GL_TEXTURE_CUBE_MAP_ARRAY of depth cubes, MAX_SHADOWS = 16 cubes, LAYER_COUNT = 96, FACE_SIZE (preset). Face f of slot i = array layer i*6+f. Format DEPTH_COMPONENT32F, LINEAR, COMPARE_MODE NONE, seamless. Shader samples with worldDir(lightPos->receiver)+slot and compares against the DOMINANT-AXIS perspective depth (NOT euclidean length — that yields a 6-point star).
- IRLShadowQuality enum (pointFaceSize, spotTileSize): LOW(256,512), MEDIUM(512,1024)=default, HIGH(1024,2048), ULTRA(2048,4096). VRAM ~40MiB / ~160MiB / ~640MiB / ~2.5GiB. applyFromSetting maps the 0..3 config to a preset; switching frees+reinits both textures lazily (setFaceSize/setTileSize).
