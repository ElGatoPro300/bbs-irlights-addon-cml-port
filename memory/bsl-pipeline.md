BSL Shaders v10 Series (by Capt Tatsu — the ancestor lineage of Complementary; structure near-identical to ComplementaryReimagined) base pipeline the IRLite inject is being PORTED into, fourth target pack after IterationRP, Photon, ComplementaryReimagined. Parent: [[MEMORY]]. Closest analog of this doc: [[complementary-pipeline]] (read both — deltas are listed against it). Port SOURCE inject = Shadres/Modification/ComplementaryReimagined/shaders/lib/irlite/irlite_lights.glsl (most recent, outline inlined, all bugfixes). Pristine base = Shadres/Original/BSL; working copy = Shadres/Modification/BSL ([[sync-workflow]]). GPU contracts unchanged: SSBO binding 7 [[addon-light-buffer-ssbo]], samplers [[addon-iris-integration]]. All anchors below verified EXACTLY-ONCE against pristine 2026-06-12.

PACK LAYOUT (verified 2026-06-12):
- Real source in shaders/program/*.glsl — COMBINED files gated #ifdef FSH / #ifdef VSH (NOT FRAGMENT_SHADER/VERTEX_SHADER like Complementary).
- world0/world1/world-1 thin wrappers: #version 120 + #extension GL_ARB_shader_texture_lod : enable + #define OVERWORLD|NETHER|END + #define FSH|VSH + include. NO program-identity defines in wrappers (BIG delta vs Complementary) -> IRLite introduces its OWN discriminators: IRLITE_SURFACE_PASS (defined by the forwardLighting hook), IRLITE_COMPOSITE_PASS (defined by the composite.glsl hook), IRLITE_NONTERRAIN (per-program ops), IRLITE_SKIP (dh programs).
- Settings #defines in lib/settings.glsl (every program includes it first). ALPHA_BLEND default 0.
- NO uniforms.glsl — uniforms declared PER-PROGRAM (delta!). AUDIT RESULT: all 8 relevant surface includers declare cameraPosition + gbufferModelViewInverse in the FSH half; gbuffers_textured FSH does NOT declare gbufferModelView -> IRLite specular must work in WORLD space (only Inverse). composite.glsl declares everything the VL/outline halves need (depthtex0/1, gbufferProjection+Inverse, gbufferModelViewInverse, cameraPosition, frameCounter, near/far, isEyeInWater, viewWidth/viewHeight) — lib must NOT redeclare these (duplicate-declaration error), only USE them.
- CRLF: BSL source files are CRLF (Photon/IterationRP/Complementary were all-LF!) -> .irlpatch anchors MUST be SINGLE-LINE (a multi-line anchor's \n never matches raw \r\n bytes — see [[patcher]] newline semantics). Applied output gets LF-inserted lines in CRLF files — validate with git -c core.autocrlf=false diff --no-index --ignore-cr-at-eol.
- #version 120 + legacy GLSL (varying/texture2D/gl_FragData). Iris transpiles all shaders to 330 core — the same mechanism that made Complementary's in-file #extension at #version 130 work. Pack itself mixes styles (voxy_opaque uses layout(location) outs). Keep the in-file #extension idiom (SSBO + cube_map_array + texture_gather). NO in-pack SSBO precedent (delta vs Complementary — its blockDataBuffer proved the idiom) -> first compile is the #1 risk, front-loaded to Phase 1.

FORWARD SHADING: lib/lighting/forwardLighting.glsl void GetLighting(inout vec3 albedo, out vec3 shadow, vec3 viewPos, vec3 worldPos, vec3 normal, vec2 lightmap, float smoothLighting, float NoL, float vanillaDiffuse, float parallaxShadow, float emission, float subsurface, float basicSubsurface) included by 12 programs: gbuffers_terrain/entities/entities_glowing/block/hand/water/textured/basic + dh_terrain + dh_water + voxy_opaque + voxy_translucent. ONE hook covers all (translucents, particles, hand, leashes incl.).
- albedo at entry is LINEAR (programs do albedo.rgb = pow(albedo.rgb, vec3(2.2)) before the call); programs sqrt-encode for storage AFTER (#if ALPHA_BLEND == 0 albedo.rgb = sqrt(...)) and composite squares back. So IRLite surface light colour = RAW LINEAR (NO pow(1/2.2) — delta vs Complementary's gamma-space surface half; same as Photon).
- worldPos = ToWorld(viewPos) = PLAYER SPACE (camera-relative world-oriented; lib/util/spaceConversion.glsl). fragWorld = worldPos + cameraPosition. normal param = VIEW-space.
- GBUFFERS_HAND PASSES normal = vec3(0.0) (literal, gbuffers_hand.glsl GetLighting call) -> lib needs a degenerate-normal fallback: dot(n,n)<0.25 -> ndl=1.0 falloff-only diffuse, skip normal-offset in shadow bias, skip spec.
- smoothLighting param = color.a (vanilla AO) in terrain/water/block/voxy_opaque, 1.0 elsewhere. NO smoothness/material params reach GetLighting -> no pack-material spec coupling possible.

THE HOOK SITES (anchors verified unique-in-file):

1. DIFFUSE + SPECULAR -> lib/lighting/forwardLighting.glsl, 3 ops:
   - include: before void GetLighting( signature line — body: #define IRLITE_SURFACE_PASS + #include "/lib/irlite/irlite_lights.glsl". (Do NOT anchor the shadows.glsl include — it sits inside #if defined OVERWORLD || defined END || (defined NETHER && defined MULTICOLORED_BLOCKLIGHT).) NOTE: the actual signature line has a TRAILING SPACE after `vec3 normal, ` — splice anchors verbatim in Phase 5.
   - compute: before albedo *= max(sceneLighting + blockLighting + emissiveLighting + nightVisionLighting + minLighting, vec3(0.0)); — body saves vec3 irliteAlbedo = albedo; + runs the merged irlite_lightSurface loop -> irliteDiffuse/irliteSpec. (Beware the commented //albedo = vec3(0.5); line right above — anchor the real line.)
   - add: after albedo *= vanillaDiffuse * smoothLighting * smoothLighting; — body: albedo += irliteAlbedo * (IRLITE_INTENSITY * irliteDiffuse) + irliteSpec;. Placement AFTER the vanillaDiffuse/smoothLighting multiply = pack vanilla-AO/fake-directional do NOT modulate IRLite light (consistent with Photon/Complementary). Pack DESATURATION applies after to everything incl. ours — accepted. Spec is additive not albedo-multiplied (parity with all ports).
   - in scope at both anchors: viewPos, worldPos, normal, lightmap, albedo, cameraPosition, gbufferModelViewInverse (NOT gbufferModelView — textured lacks it).
   - SPECULAR delta: pack GGX is UNUSABLE per-light — GGX(normal, viewPos, smoothness, baseReflectance, sunSize) hardcodes global sun lightVec + lib/surface/ggx.glsl is included AFTER forwardLighting in every program (its functions undeclared at our parse point). -> lib carries SELF-CONTAINED world-space spec: irlite_-prefixed copies of BSL's pure helpers (GGXTrowbridgeReitz / SchlickGGX / SphericalGaussianFresnel are global-free — copy + rename to avoid redefinition), real half-vector NoH (not the sun area-light NoHSquared), f0 = vec3(0.04), smoothness from an IRLITE slider (no material coupling), all vectors world-space via gbufferModelViewInverse only.
   - SPECULAR availability fact (user-confirmed in-game 2026-06-12): BSL has NO hardcoded solid-surface specular — the pack's GetSpecularHighlight on terrain/entities/hand/block is gated (OVERWORLD || END) && ADVANCED_MATERIALS && SPECULAR_HIGHLIGHT > 0 AND needs a PBR resource pack (no spec map -> smoothness 0 -> GGX early-out); only WATER hardcodes smoothness 0.9 / f0 0.02 (gate SPECULAR_HIGHLIGHT == 2, no AM); nether has no pack specular at all.
   - SPECULAR TARGET RULE (user verdict 2026-06-12: slider-spec on terrain is WRONG — "незер не должен отражать"; vanilla blocks are matte): irlite_lightSurface inks specular ONLY when nonTerrain (entities/hand/block-entities — the cinematic subjects, slider response). Terrain NEVER gets IRLite spec (even under AM+PBR — minor loss, keeps the rule simple); diffuse unaffected. nonTerrain folds compile-time -> terrain programs drop the branch. Coupling to real material smoothness instead was rejected: vanilla textures have smoothness 0, it would kill spec on BBS models too. Possible follow-up if asked: water/glass glints via an IRLITE_GLOSSY define op in gbuffers_water (the pack itself treats water as smoothness 0.9).

2. ENTITY/PROGRAM FLAGS — tiny after "#ifdef FSH" ops (#ifdef FSH verified once per file in all 6 targets):
   - #define IRLITE_NONTERRAIN in program/gbuffers_entities.glsl, gbuffers_entities_glowing.glsl, gbuffers_hand.glsl, gbuffers_block.glsl (4 ops) — drives the entities-only lightMask gate (BBS replay model = entities, model-block = block, held = hand).
   - #define IRLITE_SKIP in program/dh_terrain.glsl, dh_water.glsl (2 ops) — DH programs have NO distinguishing define of their own (DISTANT_HORIZONS is global-when-DH-loaded, can't discriminate) and DH fragments are far-LOD, pointless for near-camera lights.
   - voxy_opaque/voxy_translucent self-define VOXY_PATCH at FSH top (verified) — no op needed.
   - lib surface gate: #if defined IRLITE_SURFACE_PASS && !defined VOXY_PATCH && !defined IRLITE_SKIP.

3. VOLUMETRIC + OUTLINE. REWORKED 2026-06-12 (perf): the VL MARCH lives in the ADDED reduced-res pass program/deferred2.glsl (+6 wrappers, IRLITE_VL_PASS, writes colortex10 RGB16F sized by IRLITE_VL_RESOLUTION via size.buffer — viewport shrinks with the only render target, the Photon c0_vl mechanism; pass program-toggled by IRLITE_VOLUMETRIC; depth there = opaque depthtex0, exactly the march end we want). program/composite.glsl (the FIRST composite) hosts the OUTLINE half and the VL UPSAMPLE+ADD. NOT composite1: program.world0/composite1.enabled=LIGHT_SHAFT — composite1 is OFF for MINIMUM/LOW/MEDIUM profiles and whenever the user disables Light Shafts; composite always runs. 2 composite ops:
   - include: after #include "/lib/atmospherics/waterFog.glsl" — body: #define IRLITE_COMPOSITE_PASS + include.
   - block: AFTER vec3 reflectionColor = pow(color.rgb, vec3(0.125)) * 0.5; (Phase-3 final decision; the earlier darknessLightFactor-line plan was dropped) — past the reflectionColor capture our terms stay OUT of the prev-frame reflection colour, exact parity with the pack's own vl (added next pass). Color is LINEAR there (color.rgb *= color.rgb linearization at the ALPHA_BLEND==0 block ~L321; that text appears TWICE in the file — water-fog block too — never anchor it). Body = VL block (Phase 3, landed) + outline ink block (Phase 4.5, raw LINEAR ink — delta vs Complementary's gamma pre-pow ink) in the same body. In-scope/used: texCoord, z1, ToNDC/ToWorld, Bayer8 (dither.glsl include is unconditional), frameCounter (+#ifdef TAA temporal offset), isEyeInWater, cameraPosition. noisetex is LIGHT_SHAFT-gated — never reference it. MCBL_SS declares a dither local right below the hook — keep irlite names prefixed.
   - in scope: z0/z1, viewPos (z0-based, DH/Voxy-aware), ToNDC/ToWorld (spaceConversion included unconditionally), Bayer8 (dither.glsl included unconditionally), isEyeInWater, frameCounter, far/near, gbufferProjectionInverse, cameraPosition. noisetex is declared ONLY #ifdef LIGHT_SHAFT -> VL dither = Bayer8 (+frameCounter temporal offset), NEVER noisetex (duplicate-declaration trap).
   - VL marches to OPAQUE depth: build own viewPos1 from z1 via ToNDC (beams continue behind translucents/BBS models — translucent-phase models are in z0 only). Underwater parity: dim at isEyeInWater==1, zero at ==2. Hand pixels: z0 < 0.56 = hand (pack idiom hand = float(z0 < 0.56) in the LIGHT_SHAFT block) — skip outline there.
   - VL piggyback on the pack path REJECTED: pack vl = pow(0.25)-encoded colortex1 -> composite1 4-tap blur -> ^4*0.125 — (a+b)^4 is non-separable + the pass is LIGHT_SHAFT-gated.
   - OUTLINE deltas vs Complementary: NO normal gbuffer by default (colortex6 EncodeNormal only under ADVANCED_MATERIALS && REFLECTION_SPECULAR) -> derivative normal (dFdx/dFdy of reconstructed player pos + camera-facing flip) is the PRIMARY path, not fallback. NO universal entity mask: TAA_SELECTIVE writes entityMask 0.25-entity/0.0-blobshadow+shulker to colortex3.b from entities/hand only (terrain coverage of colortex3 unverified) -> outline TARGET=entities is BEST-EFFORT (investigate in phase; v1 default TARGET=all). Translucent-phase blindness solved by construction (composites run after all gbuffers — the Complementary bugfix-3 lesson is pre-applied by hosting in composite).
   - NAME COLLISION: BSL has its OWN "OUTLINE" feature (black pixel outline: OUTLINE_ENABLED, screen.EXTRAS, lib/post/outline.glsl, composite.glsl blocks). IRLITE_OUTLINE_* defines don't clash, but label the screen "IRLite Rim Outline" and don't anchor near the pack's outline blocks.

4. SETTINGS -> shaders.properties + lang/en_US.lang (capital US):
   - iris.features: iris.features.optional=CUSTOM_IMAGES FADE_VARIABLE exists ONLY inside #ifdef MULTICOLORED_BLOCKLIGHT (no SSBO!) -> TWO ops: replace that line appending ` SSBO`; insert before the #Multi-Colored Blocklight Prerequisites comment line: #ifndef MULTICOLORED_BLOCKLIGHT + iris.features.optional=SSBO + #endif (+ trailing blank). Exactly one iris.features line active in either branch. Binding 7 free (no bufferObject.* anywhere; MCBL uses custom images).
   - NB LATENT BUG in that 2-op recipe, found during the Solas port (2026-06-12, proven from Iris 1.7.2 source): iris.features (+ sliders/profiles/screens) are parsed from the RAW UN-preprocessed properties (ShaderProperties.java original.forEach ~L576) — duplicate key = last-in-file wins, conditionals IGNORED. So the #ifndef twin (=SSBO, lower in the file) ALWAYS overrides the CUSTOM_IMAGES FADE_VARIABLE SSBO line in the flag parse. Works today ONLY because MCBL default OFF keeps the custom-image list empty (the CUSTOM_IMAGES check passes vacuously); the moment a user enables MULTICOLORED_BLOCKLIGHT the pack throws "Custom images are being used, but the feature flag hasn't been set". FIX on next BSL touch (Modification + bsl.irlpatch regen): DELETE the #ifndef block, keep only the replaced line — the raw parse activates it in every branch (the Solas single-op recipe, see [[solas-pipeline]] hook 3). FADE_VARIABLE loss in the override is cosmetic today but also fixed by this.
   - screen: append ` [IRLITE_SETTINGS]` via replace of DYNAMIC_HANDLIGHT HALF_LAMBERT (tail of screen.LIGHTING=). IRLITE screens block inserted after the screen.AO= line. Pack screen idiom: <empty> <empty> headers, UPPERCASE names.
   - sliders: single giant sliders= line; append via replace of the unique tail pair RETRO_FILTER_DEPTH WORLD_CURVATURE_SIZE.
   - profiles (MINIMUM..ULTRA) don't touch IRLITE_* — stays independent; do NOT touch program.composite1 toggles (we don't host there).

INJECT FILE: shaders/lib/irlite/irlite_lights.glsl (+file). Program split discriminators (ALL self-made): surface half #ifdef IRLITE_SURFACE_PASS (+ !VOXY_PATCH + !IRLITE_SKIP), VL half #if defined IRLITE_VOLUMETRIC && defined IRLITE_COMPOSITE_PASS, outline half #if defined IRLITE_OUTLINE && defined IRLITE_COMPOSITE_PASS. Shadow block gate: #if defined IRLITE_SHADOWS && (defined IRLITE_SURFACE_PASS || (defined IRLITE_COMPOSITE_PASS && defined IRLITE_VOLUMETRIC && defined IRLITE_VL_SHADOWS) || (defined IRLITE_COMPOSITE_PASS && defined IRLITE_OUTLINE)) (keep the outer-IRLITE_SHADOWS escape hatch from Complementary). Composite-half-only uniforms (depth/projection/etc) must be declared INSIDE the composite gate if composite.glsl lacks them — audit says composite.glsl declares all needed; declare NOTHING that an includer already declares.

KEY DIFFERENCES vs the ComplementaryReimagined port (the porting checklist):
- Colour model: GetLighting is LINEAR -> surface light colour RAW LINEAR (drop Complementary's pow(1/2.2)); VL linear (same); outline ink linear (CHANGED from gamma). All three halves now share one colour convention — simpler.
- Specular: self-contained world-space GGX in the lib (renamed pure BSL helpers + real NoH + slider smoothness) — no pack function reuse, no gbufferModelView dependency.
- Discriminators all self-made (wrappers carry no program defines); entity flag needs 4 per-program ops; DH skip needs 2 ops (vs zero in Complementary).
- VL/outline host = composite (NOT composite1 — program-toggled off) and both halves share one hook block after one anchor.
- Hand: normal = vec3(0.0) -> degenerate-normal fallback in the surface loop.
- iris.features needs 2 ops (flag conditional + missing SSBO; Complementary needed none).
- CRLF pack -> single-line anchors only in the .irlpatch.
- Patched pack becomes Iris-only (SSBO); pristine BSL also targets OptiFine — by design, same stance as all ports.
