Когда правки меняют код/поведение, я должен ПРЕДЛОЖИТЬ бампнуть версию трилогии. Критичен ТАЙМИНГ: предлагать бамп ТОЛЬКО ПОСЛЕ того как юзер дал команду коммитить (тот же коммит-чекпоинт из [[commit-checkpoints]]). НИКОГДА не поднимать тему версии до этого — не в середине работы, не при первом показе правок, не «на всякий случай».

Why: версия привязана к коммит-чекпоинту (релиз-точке), а не к каждой промежуточной правке. Пока не решено, что это коммит, разговор о версии = преждевременный шум, сбивает фокус. Юзер сам решает, когда состояние = релизное; предложение бампа идёт ВНУТРИ момента коммита, не опережает его.

How to apply: дождаться «коммитим»/подтверждения коммита -> в сводке коммита добавить предложение бампнуть VERSION (PATCH=багфикс, MINOR=фича, MAJOR=ломка совместимости, см. [[project-versioning-lockstep]]) + строку в CHANGELOG.md; бампать только после согласия. Если юзер коммитит без бампа — не настаивать. Механизм бампа (один VERSION-файл -> build-trilogy штампует во все линии) = [[project-versioning-lockstep]].

Связь: тайминг-надстройка над [[commit-checkpoints]] (сначала «коммитим», ПОТОМ предложить версию); механизм = [[project-versioning-lockstep]].
