Полные отчёты: `_archive/shadow-refactor-research-2026-07-01/`.

IN-WORLD PASS 2026-07-02 (photon_v1.3b_IRLights): юзер подтвердил оба видимых дискриминатора по эталонной визуализации — (1) края point-теней сглажены gather'ом (лесенка осталась — это разрешение карты, но ступеньки с плавными переходами), (2) узор зерна в полутени = ровный IGN без сгустков. Значит весь файл, вкл. Ф1a-пирамиду, активен на GPU. НЕ замерено: FPS-выигрыш early-out/пирамиды (нужен GPU-bound замер), судьба «зернистого квадрата» на высокой Softness. Известный шум лога: «#endif without #if» 2x на каждую загрузку photon-пака (null:0:0, INFO сканера опций Iris, НЕ фатально — баланс директив в файле 42/42 идеален; до правок photon-загрузок в логах нет, так что pre-existing или нет — неизвестно).

СТАТУС Ф1b (2026-07-02): IN-WORLD PASS (юзер: point-тени идентичны, артефактов нет); коммит core 0f6d644. РЕАЛИЗОВАНА, ревью 3 линзами (1 блокер закрыт). Java irl-core: PointShadowPyramid — face-major GL_TEXTURE_2D_ARRAY RG32F (base=face/2, 96 слоёв layer=slot*6+face, levels=log2(FACE_SIZE)); ДВЕ compute-программы: CUBE_SRC (depth samplerCubeArray -> lod0, реконструкция направления обратной таблицей faceDir — доказана взаимная обратность с irlite_cubeFaceUV; чтение в центрах текселей = точный тексель даже при LINEAR, доказано вкл. краевые тексели при seamless) и MIP_SRC (texelFetch 2D-array); батч markDirty/flushDirty после point-цикла, layered image binding, dispatch z=6 (все грани слота), cube/2D-array байнды raw (GlStateManager их не трекает — только 2D). ShadowBaker: markDirty в 3 сайтах point-цикла (no-cache/dirty-static/overlay каждый кадр — partial copyMask ребилдит все 6 граней, консервативно ок); PointShadowArray.delete -> каскад. GLSL: early-out c edge-гардом (футпринт у границы грани -> полный PCSS: face-local пирамида не видит соседнюю грань), uvHalf = footWorld/max(zPersp-footWorld,1e-4) (консервативный знаменатель — бонд 1-го порядка эродирует у краёв), footTex В DEPTH-ТЕКСЕЛЯХ = uvHalf*faceRes (БЛОКЕР ревью: base-тексели с формулой лода спота = октава недопокрытия, до 2x при footTex чуть ниже степени двойки); textureSize-гейт base==face/2 (unbound array-сэмплер читает чужой 2D_ARRAY юнита, напр. cookie — pyrMax>0 не ловит ненулевой мусор); хелпер irlite_cubeFaceUV (спек-таблица, gather-тап отрефакторен). Миксины 4 файлов оба мода: регистрация 2D + rebind GL_TEXTURE_2D_ARRAY (паттерн cookie). СТАТУС Ф2a (2026-07-02): РЕАЛИЗОВАНА + ревью 3 линзами, 3 major математики закрыты; ждёт in-world ИНТЕРЬЕРНОГО теста (light bleeding = худший кейс Чебышёва) + свипа Softness через порог веток. Java irl-core: SpotShadowEvsm — EVSM4 RGBA32F base=atlas/2 + mips (levels=log2(TILE), trilinear сэмплер), temp (TILE/2)^2; 3 compute: CONVERT (depth -> ЛИНЕАРИЗОВАННЫЙ lz=(dist-near)/(far-near) -> warp wp=exp(42 lz), wn=-exp(-8 lz); усреднение моментов 2x2 = точный префильтр; per-tile uniform far через markDirty(tile, range) — ревью: warp на perspective z01 деградирует к plain-VSM на дали + floor растёт квадратично), BLUR (separable 1-4-6-4-1, кламп в тайл, srcLod), MIP (2x2 avg); КАЖДЫЙ mip ре-блюрится после даунсэмпла (ревью: голая avg-цепочка НЕ удваивает ширину фильтра на лод — sigma~2^L/sqrt(3) вместо 2*2^L, широкие полутени были бы в 2-3.5x резче PCF-эквивалента; с пер-лод блюром sigma~2.3*2^L, расхождение ~15%); батч по паттерну пирамиды. Сырая depth-карта живёт: blocker search/контакт-PCF/VL нетронуты. GLSL: ветка после blocker search, penDepthTex>=4 && size-гейт (evsmRes*2==atlasSize.x) -> lod=log2(pen*0.25), UV-кламп полутекселя ГРУБОГО мипа (ceil(lod), min 0.125=полтайла — tMin<=tMax всегда), 1 textureLod + двойной Чебышёв (variance floor = (производная warp * 1e-4)^2; формула irlite_chebyshev корректна для обеих компонент — оба warp возрастающие), min, linstep bleed 0.10 (ревью: 0.25 ломает бесшовность — Чебышёв ТОЧЕН для двухплоскостного распределения, EVSM==PCF до linstep; скачок 0.17 + спекл на изолинии порога), БЛЕНД-ЗОНА pen 4..6 текселей = smoothstep-mix PCF<->EVSM (early-out проба отключена в зоне — mix требует полного цикла). Миксины обоих модов: irl_spotEvsm (plain 2D). ОТКРЫТЫЕ ХВОСТЫ Ф2a: (1) overlay-путь = convert+blur+mips каждый кадр на тайл (~40-60MiB трафика/тайл MEDIUM, HIGH с 16 overlay-спотами ~8-12ms — неприемлемо; варианты: CPU max-penumbra cull по bulbSize, полноатласный temp для батч-барьеров (+64MiB), НЕ джиттер — шов веток); (2) формат-лестница fp16 на LOW/MED; (3) CN=8 можно поднимать до 42 без переполнений если дальний bleeding проявится; (4) VL всё ещё на raw depth (ESM-канал = будущая опция).

КОЗЫРЬ ПРОЕКТА (консенсус всех отчётов): карты бейкаются и кэшируются -> бейк-препроцесс (blur/mips/пирамиды) амортизирован в ноль. Ни один шейдерпак так не может (их sun-map перерисовывается каждый кадр) — префильтруемые техники у нас выгоднее, чем где-либо.

ТЕКУЩАЯ ЦЕНА (Q2 default): 10 blocker + 18 PCF тапов на (свет,пиксель); spot gather 4 сравнения/тап, point бинарный compare; ротация spatial-only; fast=false всегда в surface (LOD нет); early-out нет. Бенчмарк хост-паков: blocker 3-6 (Photon 3!), PCF 6-16, temporal-джиттер под TAA у всех VPS-паков, hardware PCF (BSL/CR/Bliss) = бесплатный x4/тап (у нас COMPARE_MODE NONE), early-out Photon по первым 4 тапам, шарпенинг суб-тексельной пенумбры (Photon/Kappa).

РЕКОМЕНДОВАННЫЙ ПУТЬ (консенсус судей; каждый пункт под своим #define/флагом, старый путь = fallback):
- Ф0 GLSL-гигиена (только irlite_lights.glsl, ноль Java): early-out по первым 4 тапам; fast=true для тусклых/дальних (attenuation-LOD); фикс двойного полного PCSS в BSL outline (irlite_outlineInk зовёт fast=false); backport VL-stride BSL->Photon; унификация края тайла (сейчас 3 политики: blocker continue / PCF lit / gather clamp — бандинг у границы конуса); gather/сглаживание для point (несогласован со spot); IGN вместо sin-hash (highp); шарпенинг узкой пенумбры (кламп к ~2 texel + linstep); бюджеты тапов к нативным (резать default почти вдвое).
- Ф1 пирамидный PCSS: на бейке min/max/avg mip-пирамида (аддитивная RG-текстура рядом с depth; семантика карт, SSBO-контракт, VL НЕ меняются) -> в кадре 1 fetch early-out вне полутени (отсекает 70-90% пар свет-пиксель), avgBlocker 1 fetch вместо 6-20 тапов (заодно стабилизирует шумную оценку penumbra), адаптивные квантованные тапы (3-4 ступени против дивергенции). Java: аллокация пирамиды, downsample после endPass, учёт в shadowBakeBudget (не усугубить спайк C10 ~320ms). Гочи: mip-clamp чтобы footprint не пересёк тайл 4x4; textureLod на #120-паках (BSL/Bliss) — непроверенная гипотеза, прототипировать ПЕРВОЙ.
- Замер после Ф1 (FrameProfiler + in-world): если «зернистый квадрат» на высокой Softness или потолок качества остаются -> Ф2.
- Ф2 EVSM/MSM-префильтр: конверсионный пасс depth->moments ПОСЛЕ бейка (кастер-шейдеры не трогаются — их 3 пути рисуют depth как сейчас) + Gaussian blur + mip-цепочка; сэмплинг = Чебышёв 1 trilinear fetch при любой Softness; contact-PCF ветка по сырой depth для честного контактного затвердевания (гибрид). Spot = EVSM4 (fp32/fp16 по пресету), point = MSM RGBA16-UNORM или EVSM fp16 (страховка от bleeding). Закрывает «зернистый квадрат» ПО ПОСТРОЕНИЮ при любой из двух конкурирующих гипотез (бейк/сэмплинг — пост-мортем исключил шейдер, но GLSL-аудит нашёл в сэмплинге статичный шум ротации как кандидата; пре-блюр глушит обе); лечит bobbing-acne (нет бинарного compare) и даль (trilinear mips = shadow-AA); VL = 1 мягкий fetch на шаг.

РИСКИ Ф2: light bleeding в Minecraft-интерьерах (мультислой пол/мебель/потолок = худший кейс Чебышёва; ранний in-world тест интерьера обязателен); память x2-4 (HIGH live+static уже ~1.25GiB D32F; нужна лестница форматов по пресетам); gutter/padding тайлов атласа = правка layout-контракта Java+GLSL зеркально (исторически хрупко); per-face блюр куба без швов; VL-контракт Photon меняется (depth->moments).

ФАКТЫ БЕЙКА (аудит Java): Softness НЕВИДИМА бейк-стороне (grep пуст, не в сигнатуре — смена слайдера не ре-бейкает, влияет только на GLSL-радиусы); формат D32F оба layout, color-attachment нет (glDrawBuffer NONE); mip нет (MAX_LEVEL=0 захардкожен); spot NEAREST vs point LINEAR (LINEAR по raw depth = интерполяция глубин, сам по себе кандидат в артефакты граней); двухслойный static/live ping-pong + copyStaticToLive = готовый каркас под blur-пасс (blur вместо копии); единые точки аллокации createAtlas/createArray.

TEMPORAL: все 7 хост-паков джиттерят под TAA (Complementary/Solas отключают без TAA через #ifdef), но канон проекта = spatial-only ротация (осознанный отказ, shader-shadow-sampling). Вводить только как opt-in #define default OFF с TAA-гейтом, после явного пересмотра инварианта.

ОТКЛОНЕНО: screen-space blur теней (агрегированный буфер = кросс-блед пенумбр разных цветных источников; расходится с VL; ни один из 8 паков так не делает); SS contact shadows (только deferred Photon/IterationRP = 2/7 паков, аддитивная стоимость 8-16 тапов, целевые артефакты не закрывает); SDF-гибрид (шов «аналитическая тень блоков vs map-тень актёров» в одном кадре, потеря cutout-детали заборов/листвы, объём = отдельный проект); SAT-VSM (только spot, усиливает несогласованность spot/point).
