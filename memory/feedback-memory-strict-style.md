При создании/правке любого файла памяти этого проекта держать строгий вид под чтение LLM (правило задано юзером 2026-06-29; тогда же переформатированы все активные файлы, см. [[reference-edit-routing-by-area]] как образец стиля):
- Ноль эмодзи и значков статуса (галочки/кресты/предупреждения и т.п.) — статус словами (DONE / OPEN / RETIRED / НЕ / отменено).
- Без декоративного жирного (**...**), блок-цитат (>), зачёркивания (~~...~~), разделителей ===.
- description = одна короткая строка-хук (без эмодзи, без абзацев).
- Хвостовую «Связь»-секцию (если нужна) свернуть в одну строку «Связь: …» + [[links]], а не шаблон «Применимость/Связанные/Источник».
- Таблицы ОСТАВЛЯТЬ (плотные данные, LLM читает хорошо) — только без эмодзи/жирного внутри ячеек.
- Сохранять как есть: frontmatter (name/metadata), код-блоки, [[links]], структуру MEMORY.md (одна строка на память).

Why: декор раздувает токены и мешает чтению; цель — оптимизировать чтение/работу LLM.
How: писать факты терсово, форму держать плоской. Это переформат ФОРМЫ, а не вычистка содержания — факты не удалять (для слияния дублей есть отдельный проход consolidate-memory).
