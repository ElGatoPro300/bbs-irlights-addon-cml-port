IRLite development sync workflow — directory roles and the edit/test loop. Parent: [[MEMORY]]. The .irlpatch format/applier this feeds into: [[patcher]]. Shader docs describe the END STATE of the inject ([[shader-iterationrp-pipeline]] etc.); THIS doc describes how edits flow through folders during active development. All paths under the repo root C:\Users\Qualet\Documents\Project\Minecraft\BBS\bbs-irlights-addon.

DIRECTORY ROLES (do not confuse them). REFACTORED 2026-06-11: pristine packs moved under Shadres\Original\<pack>, and the working copy is now per-pack under Shadres\Modification\<pack> (was a flat Shadres\Modification). Both Original and Modification hold one subfolder per shaderpack.
GIT 2026-06-12: the whole Shadres\ tree is gitignored and UNTRACKED (commit e6893eb) — third-party packs are not redistributable, and the repo license (CC BY-ND 4.0) covers IRLite code only. Shader edits in Modification are NOT version-controlled anymore; the durable artifact is patches\*.irlpatch. Pre-untrack Shadres history survives in commits before e6893eb.
- Shadres\Original\<pack>  — PRISTINE pre-patch shader library, one folder per pack (currently Shadres\Original\IterationRP and Shadres\Original\Photon). Read-only reference for code analysis (understanding the base pipeline) AND the diff baseline the patcher diffs/copies from. NEVER edit here.
- Shadres\Modification\<pack>  — the shaders being actively modified RIGHT NOW with the IRLite integration injected, one folder per pack (currently Shadres\Modification\Photon, which holds LICENSE + shaders\). All hand-edits happen here.
- patches\  — finished .irlpatch files (iterationrp.irlpatch, photon.irlpatch). A patch is regenerated by diffing Modification\<pack> against the matching pristine Shadres\Original\<pack> ONLY when a modification is confirmed working, and ONLY on the user's explicit command. Do not auto-regenerate the patch.
- run\shaderpacks\  — the dev Minecraft instance's shaderpacks (IterationRP, IterationRP_IRLite, Photon_IRLite, ...). This is what the running game actually loads.

EDIT -> TEST SYNC LOOP (during testing, before the patch is made):
1. Hand-edit shader files in Shadres\Modification\<pack> (and/or edit the Java addon side).
2. Copy the changed shader file(s) from Shadres\Modification\<pack>\shaders into run\shaderpacks (OVERWRITE the matching files in the active pack). This is a manual file copy, not the patcher — the patcher (.irlpatch apply) is only used to produce the distributable patched pack once the change is final.
3. Then either:
   - Java/addon side changed -> run the Fabric dev client (gradle runClient) so the recompiled mod loads.
   - Only shader changed -> hot-reload shaders in the already-running dev instance (Iris shader reload, no MC restart needed).

KEY DISTINCTION: the test-time copy (Modification\<pack> -> run\shaderpacks) is a fast manual sync for iteration. Turning the change into patches\*.irlpatch (diff Modification\<pack> vs pristine Shadres\Original\<pack>) is a SEPARATE, deliberate step done on user command once the modified shader is verified correct — see the anchor-capture rules in [[patcher]]. Keep the shader-*.md docs ([[shader-iterationrp-pipeline]], [[shader-irlite-glsl]], [[shader-shadow-sampling]], [[shader-volumetric]], [[shader-settings]]) in sync when a Modification edit is promoted to a patch.
