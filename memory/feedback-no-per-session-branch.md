Не создавать отдельную git-ветку под каждую сессию или задачу. Работать и коммитить в текущей ветке (main или той, что выбрана пользователем). Новую ветку заводить только по явной просьбе.

Why: пользователь сам управляет ветвлением; авто-ветки под сессию плодят мусорные ветки и worktree, которые потом приходится чистить.
How: в начале задачи не выполнять git checkout -b и не создавать worktree; редактировать в текущей ветке. Даже если задача крупная — не ветвиться без запроса. Маршрутизация: [[reference-edit-routing-by-area]].
