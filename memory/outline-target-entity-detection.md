Фича 2026-06-30: к старому outline ([[project-photon-outline-switch-to-old]]) добавлена опция IRLITE_OUTLINE_TARGET // [0 1 2] (0 both / 1 only entities / 2 only blocks) — те же коды, что lightMask в SSBO (0 all/1 entities/2 blocks). Гейт ставится в composite-проходе, где зовётся irlite_outlineInk: outline рисуется только если пиксель проходит target-фильтр по per-pixel entity-метке из gbuffer пака. Паттерн гейта (compile-time, не входим в outline зря): bool irliteODraw=<z-gate>; #if IRLITE_OUTLINE_TARGET!=0 { читаем метку; #if==1 draw=isEntity #else draw=!isEntity } #endif; if(irliteODraw){...outline...}. GUI: IRLITE_OUTLINE_TARGET в screen.IRLITE_OUTLINE_SETTINGS сразу за IRLITE_OUTLINE + в sliders= (в середину, хвост остаётся IRLITE_OUTLINE_GLOW_STRENGTH — иначе падают slBody.EndsWith-ассерты генераторов) + lang option/comment/value.0-2 (Solas: en_US+ru_RU).

Ключ: BBS-морфы (актёры) рендерятся entity-путём и получают ДЕФОЛТНУЮ entity-метку пака (НЕ PBR-специфичную) — детектор обязан ловить дефолт, иначе морфы не распознаются (симптом: Only Entities пусто, Only Blocks рисует и на entity). Это была первая ошибка на Complementary (ловил только PBR 100..199).

Per-pack entity-метка (в composite-проходе):
- Complementary (done): materialMask = colortex6.g*255. entity = (100..199) ИЛИ ==254 (дефолт entity/hand, сюда же BBS-морфы); block/terrain дефолт 0. gbuffers_entities:143 дефолт 254, +100 только при PBR (!=254). colortex6 пишется всегда, не перезаписывается до composite1. Тест: irliteOMat>99.5 && <199.5 || >253.5.
- Solas (done): РОДНАЯ метка — gbuffers_entities пишет colortex3=(0,0,0,1) безусловно (нормали закомментированы), terrain пишет encodeNormal(n)=n.xy/f+0.5 (.xy≳0.15, центр 0.5). colortex3=RGBA16, не перезаписан до composite1, уже объявлен. Тест: colortex3.xy.x<0.1 && .y<0.1 = entity. Без инжекта.
- BSL (done, ИНЖЕКТ): нет готовой метки в стоке (colortex1=RGB8 без alpha; родной colortex3.z=entityMask пишется лишь при TAA_SELECTIVE/ADVANCED_MATERIALS — оба off по умолчанию; hand=z0<0.56 чисто depth). Инжект: gbuffers_entities base DRAWBUFFERS:01->013 + gl_FragData[2]=vec4(0,0,entityMask,1) (entityMask=0.25 для непрозрачных, стр ~182); composite +uniform sampler2D colortex3 + читаем .z>0.1. terrain НЕ трогаем (colortex3 clear=0 -> не-entity). Под-ветки MCBL/PBR re-pin свой layout (последний DRAWBUFFERS выигрывает — паттерн пака). Caveat: конфиг MCBL_SS-only (без TAA/ADV) метку не пишет -> target не сработает (MCBL off по умолчанию, приемлемо). gen-bsl-patch.ps1 ОБНОВЛЁН: +2 операции (composite after "uniform sampler2D colortex1;"; gbuffers_entities replace "/* DRAWBUFFERS:01 */"->013 + after FragData[1]).
- Bliss (PENDING, сложнейший): material в jodie-2x8 colortex1.a, outline видит decodeVec2Y -> hand=0.75; raw MATERIALS (all_translucent.fsh:468-484) 1.0 water/0.9 entity/0.8 refl-entity/0.7 refl-block/0.1 hand; зависит от TRANSLUCENT_ENTITIES (если off — морфы через all_solid, colortex1.a не обновляют). Нужен инжект или доказать рабочий канал в рантайме.
- RethinkingVoxels (done+committed 0f14cd3): Complementary-форк — тот же materialMask (colortex6.g RGBA8, gbuffers_entities дефолт 254 +100 PBR), гейт в program/composite.glsl (нет composite1), texelFetch(colortex6,texelCoord,0).g; gen-rethinkingvoxels не менялся (cOutline берёт весь блок). VL=colortex15 ([[rethinkingvoxels-pipeline]]).

Диагностика: grep TARGET в run/shaderpacks/<pack>_IRLights (applied) vs patches/<pack>.irlights vs jar — расхождение = stale working copy. runClient грузит мод из build/classes+build/resources (sourceSet), НЕ из dev jar; processResources кэширует patches/ (нужен --rerun-tasks); финальный jar = remapJar (build/libs), не jar task.
